KanjiTest.Online — Project Summary
====================================
Platform: https://www.kanjitest.online
GitHub:   https://github.com/xh8khar/kanjitest.online
Domain:   www.kanjitest.online (Cloudflare DNS → GitHub Pages)

Stack
-----
- Next.js 16.2.10 (App Router, static export to out/)
- TypeScript + Tailwind CSS v4
- Zen Maru Gothic (Google Fonts)
- Cream (#fdf8f0) background, ink (#1a1a1a) text
- No backend, no DB, no auth — React state + localStorage only

Status
------
- N5: 79 kanji, 237 examples, 219 vocabulary words, 20 test sets, flashcards ✓
- N4: 171 kanji, 513 examples, 473 vocabulary words, 20 test sets, flashcards ✓
- N3: 367 kanji, 1101 examples, 935 vocabulary words, 20 test sets, flashcards ✓
- N2/N1: hub pages only ("Coming Soon")

Features per Level
------------------
- Study: Browse kanji with kun/on readings, example words, sentences
- Flashcards: Flip through kanji, mark "Know it" / "Again" (localStorage)
- Vocabulary: Words with readings, meanings, kanji badges, sentences
- Test: 20 sets × 20 questions, alternating kanji→reading / reading→kanji, instant MC feedback

Pages Generated
---------------
- 2947 static pages total (N5 + N4 + N3 study, flashcards, tests, vocabulary)
- All URLs use kanji characters for study/flashcards (SEO-optimized)
- Numeric IDs for test sets (meaningful identifiers)

Key Decisions
-------------
- Brand: "KanjiTest.Online"
- Logo: Black circle with 日
- Template-based title generation ("%s | KanjiTest.Online") in root layout
- output: "export" only in production (NODE_ENV === "production")
- Kanji IDs per level: N5=1-79, N4=80-250, N3=1-367
- Study/flashcard URLs use kanji char: /n5/study/一/ instead of /n5/study/1/
- Vocabulary slugs use hiragana reading: /n5/vocabulary/あおい/
- Separate localStorage keys per level for flashcards progress
- Homepage modeled after kanji123.org: hero + level cards + content sections
- Mobile-first responsive design, horizontal chip nav on mobile

Recent Changes (July 3, 2026)

Build
